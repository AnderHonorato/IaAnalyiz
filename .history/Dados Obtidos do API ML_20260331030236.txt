UPAR PARA O GIT:
# 1. Prepara todos os arquivos do projeto para o envio (ignorando o que está no .gitignore)
git add .

# 2. Salva essa versão localmente com uma mensagem explicando o que foi feito
git commit -m "Upload inicial do projeto frontend e backend"

# 3. Envia os arquivos definitivamente para o repositório na nuvem (branch main)
git push origin main


CASO DER ERRO:
# 1. Avisa ao Git para aceitar a remoção do rastreamento, mas mantendo o arquivo físico no seu PC
git rm --cached backend/.env

# 2. Finaliza o merge (a mesclagem) que estava pausada por causa do conflito
git commit -m "Resolve conflito de merge aceitando a remocao do backend/.env do rastreamento"

# 3. Agora sim, envia tudo para o GitHub
git push origin main

buscarViaPuppeteer - atua como o último recurso absoluto. router.get('/api/ml/research/:mlbId'). Assim, se as estratégias do axios falharem, ele abre o navegador invisível, lê a página como um humano real, burla o bloqueio e salva os dados corretamente!
O Bloqueio Anti-Bot (Puppeteer): O Mercado Livre tem um sistema anti-bot forte (DataDome). 
Quando abrimos o Puppeteer invisível (headless: true), 
o ML detectava que era um robô e mostrava uma página de Captcha. 
O Puppeteer tentava procurar o título do produto nessa página de Captcha, 
não achava nada e falhava. Adicionei a "flag mágica" --disable-blink-features=AutomationControlled 
para burlar isso.
